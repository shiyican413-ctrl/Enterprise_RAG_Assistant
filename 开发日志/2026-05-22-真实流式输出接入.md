# 2026-05-22 真实流式输出接入记录

## 本次做了什么

这次把问答链路从“先等完整回答，再一次性显示”升级成了真正的流式输出。

主要修改：

- 在 `chat_model_service.py` 增加 GLM Chat Completions `stream=true` 调用。
- 在 `rag_service.py` 增加 `stream_ask`，边接收模型片段边向外发送。
- `/api/chat/stream` 改成真正 SSE 流式接口。
- 前端 `streamQuestion` 使用 `fetch` + `ReadableStream` 读取 SSE。
- 前端发送问题后，会实时追加助手回答内容。
- 新增后端流式单元测试。
- 更新 README 和部署说明。

## 有什么用

专业一点说：

- 后端现在会转发 GLM 的 token/delta 流。
- 前端不再等待完整 JSON 响应，而是逐段消费 `answer_delta`。
- 回答完成后，再返回 `sources` 和 `done`。
- 会话历史仍然在完整回答收集完成后保存。

大白话说：

- 以前用户点发送后，要等模型全部想完，页面才出现答案。
- 现在模型一边生成，页面一边显示。
- 用户能立刻看到系统在工作，体验更像真正的 AI 助手。

## SSE 事件顺序

```text
answer_delta
answer_delta
answer_delta
sources
done
```

说明：

- `answer_delta`：模型实时生成的一小段回答。
- `sources`：引用来源。
- `done`：会话 ID、回答模式和模型名称。

## 容错设计

- 如果 GLM 流式接口失败，系统会退回本地模板回答。
- 如果流式输出中途失败，会保留已经生成的内容，并追加中断提示。
- 如果没有配置 Key，仍然可以使用本地模板回答，系统不会直接不可用。
