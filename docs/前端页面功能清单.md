# Enterprise RAG Assistant 前端页面功能清单

本文档用于拆分前端页面范围，明确每个页面需要放哪些内容、承担哪些功能，方便后续按模块开发和验收。

## 1. 登录页

页面目标：完成用户身份识别，进入企业知识库系统。

页面内容：

- 系统名称：企业 RAG 智能问答平台
- 登录表单：账号、密码
- 登录按钮
- 登录状态提示：账号错误、密码错误、服务不可用
- 可选能力：记住登录状态、忘记密码、企业 SSO 登录入口

核心功能：

- 调用登录接口完成身份认证
- 登录成功后进入工作台或智能问答页
- 根据用户角色加载对应菜单权限
- 未登录访问受保护页面时跳转回登录页

## 2. 工作台首页

页面目标：展示系统整体状态，让用户快速进入常用功能。

页面内容：

- 顶部状态区：后端服务状态、当前用户、当前角色
- 核心指标：文档数量、知识片段数量、今日问答次数、引用命中次数
- 快捷入口：智能问答、上传文档、知识库管理、会话记录
- 最近动态：最近上传文档、最近问答、最近索引任务
- 系统能力概览：文档入库、语义检索、流式回答、引用追溯

核心功能：

- 获取系统健康状态
- 获取知识库统计信息
- 展示最近操作记录
- 为常用页面提供快速跳转入口

## 3. 智能问答页

页面目标：提供企业知识库问答体验，是系统的核心使用页面。

页面内容：

- 对话消息区：用户问题、助手回答、生成状态
- 问题输入框：支持 Enter 发送、Shift + Enter 换行
- 发送按钮
- 新建对话按钮
- 回答模式切换：快速模式、思考模式
- 当前会话 ID 或会话状态
- 常用问题示例：如报销、请假、制度查询
- 回答免责声明：AI 生成内容需结合引用核验

核心功能：

- 向 `/api/chat/stream` 发起流式问答请求
- 展示打字机式流式输出
- 保存并延续多轮会话上下文
- 支持新建会话
- 显示回答所使用的模型和模式
- 问答失败时展示错误提示

## 4. 引用来源面板

页面目标：让用户核验回答依据，降低模型幻觉风险。

页面内容：

- 当前回答引用数量
- 引用文档名称
- 引用片段内容
- 匹配度分数
- 片段序号
- 可选能力：展开全文、跳转原文、复制引用

核心功能：

- 接收问答接口返回的 sources
- 按匹配度展示引用来源
- 将引用来源与当前回答关联
- 支持用户查看证据片段

说明：当前代码中该能力放在右侧“知识库面板”的“证据来源”区域，后续可以独立成抽屉或详情页。

## 5. 知识库管理页

页面目标：管理企业文档和知识库索引。

页面内容：

- 知识库统计：文档总数、片段总数、索引状态
- 文档列表：文档名称、片段数、上传时间、处理状态
- 操作按钮：刷新、删除、重新入库、重建索引
- 搜索与筛选：文档名称、文件类型、上传人、处理状态
- 空状态：暂无文档时提示上传企业资料

核心功能：

- 调用 `/api/documents` 获取文档列表
- 调用 `/api/documents/{document_id}` 删除文档
- 调用 `/api/knowledge/rebuild` 重建知识库
- 展示文档处理状态和入库结果
- 后续接入文档元数据和权限过滤

## 6. 文档上传页

页面目标：完成企业资料上传、解析和入库。

页面内容：

- 文件选择区
- 支持格式说明：txt、md、csv、json、pdf
- 上传按钮
- 入库进度：上传中、解析中、切分中、向量化中、完成、失败
- 上传结果：文档名称、片段数量、错误原因
- 可选元数据：所属部门、文档类型、标签、访问权限

核心功能：

- 调用 `/api/documents/upload` 上传文件
- 上传成功后刷新文档列表
- 上传失败时展示错误信息
- 后续支持批量上传、拖拽上传、重复文件检测

说明：当前代码中该能力已经放在右侧“知识库面板”的“文档入库”区域。

## 7. 会话记录页

页面目标：查看和管理历史问答记录。

页面内容：

- 会话列表：标题、最近提问、更新时间、消息数量
- 会话详情：完整问答消息、引用来源、模型模式
- 搜索框：按问题关键词搜索
- 操作按钮：继续对话、重命名、删除、收藏、导出
- 空状态：暂无会话记录

核心功能：

- 按用户获取历史会话列表
- 调用 `/api/chat/conversations/{conversation_id}` 获取会话详情
- 从历史会话恢复上下文继续提问
- 后续支持收藏、导出、按部门或业务分类

## 8. 权限审计页

页面目标：展示用户权限、知识库访问范围和关键操作日志。

页面内容：

- 用户列表：姓名、账号、角色、状态
- 角色列表：管理员、普通用户、知识库维护人员
- 权限范围：可访问知识库、可上传、可删除、可审计
- 操作日志：登录、上传、删除、重建索引、问答查询
- 筛选条件：用户、时间、操作类型、结果状态

核心功能：

- 展示不同角色的访问边界
- 查询关键操作日志
- 支持管理员审计文档和问答行为
- 后续由 Django 业务后台提供用户、角色和日志接口

## 9. 模型配置页

页面目标：管理大模型、Embedding 模型和检索参数。

页面内容：

- Chat 模型配置：快速模式模型、思考模式模型
- Embedding 模型配置：模型名称、向量维度、服务地址
- API Key 状态：已配置、未配置、不可用
- 检索参数：top_k、相似度阈值、是否启用 rerank
- 回答参数：温度、最大输出长度、系统提示词
- 测试按钮：测试连接、测试 Embedding、测试问答

核心功能：

- 展示当前模型配置
- 修改模型服务参数
- 检查模型服务可用性
- 后续支持 OpenAI Compatible API、Ollama、DashScope、GLM 等适配

## 10. Agent 工具页

页面目标：管理后续可被 Agent 调用的企业业务工具。

页面内容：

- 工具列表：工具名称、描述、接口地址、启用状态
- 工具类型：订单查询、库存查询、假期余额、报销进度、工单创建
- 参数说明：入参字段、出参字段、鉴权方式
- 测试面板：输入测试参数并查看返回结果
- 调用日志：调用时间、调用人、结果状态

核心功能：

- 新增和维护工具配置
- 启用或停用工具
- 测试业务 API 连通性
- 为 LangGraph 或 Agent 服务提供工具注册信息

## 11. 系统设置页

页面目标：集中管理系统级开关和运行配置。

页面内容：

- 基础信息：系统名称、版本号、后端地址、数据目录
- 文件设置：允许格式、单文件大小限制、上传目录
- 索引设置：切分长度、重叠长度、向量库类型
- 安全设置：登录有效期、访问控制、审计开关
- 运维设置：健康检查、清理缓存、重建索引

核心功能：

- 查看当前系统配置
- 修改可变配置项
- 触发维护类操作
- 展示配置保存结果和错误信息

## 12. 帮助中心页

页面目标：帮助用户理解系统使用方式和常见问题。

页面内容：

- 使用流程：上传文档、等待入库、发起提问、核验引用
- 支持文件格式说明
- 常见问题：无引用、回答不准确、上传失败、服务离线
- 示例问题库：制度查询、产品资料、客服辅助
- 联系管理员入口

核心功能：

- 提供静态帮助文档
- 引导用户排查常见问题
- 后续可接入企业内部帮助文档知识库

## 13. 管理后台页

页面目标：给管理员提供集中管理入口。

页面内容：

- 用户管理
- 角色权限管理
- 文档元数据管理
- 操作日志管理
- 模型配置管理
- 系统运行状态

核心功能：

- 聚合管理类页面入口
- 展示管理员专属能力
- 后续与 Django Admin 或自研后台对接

## 页面优先级建议

第一阶段优先完成：

1. 智能问答页
2. 文档上传页
3. 知识库管理页
4. 引用来源面板
5. 会话记录页

第二阶段补齐：

1. 登录页
2. 工作台首页
3. 权限审计页
4. 模型配置页
5. 系统设置页

第三阶段扩展：

1. Agent 工具页
2. 帮助中心页
3. 管理后台页

## 当前代码对应关系

当前 `frontend/next-web/app/page.tsx` 已经把多个核心能力合并在一个页面中：

- 左侧 `Sidebar`：核心菜单、项目能力、服务状态
- 中间 `ChatPanel`：智能问答、流式输出、回答模式、新建会话
- 右侧 `InspectorPanel`：文档上传、文档列表、引用来源、处理链路

后续如果要做成正式多页面结构，可以按本文档将现有单页拆成独立路由，例如：

- `/dashboard`：工作台首页
- `/chat`：智能问答页
- `/knowledge`：知识库管理页
- `/knowledge/upload`：文档上传页
- `/conversations`：会话记录页
- `/audit`：权限审计页
- `/settings/models`：模型配置页
- `/settings/system`：系统设置页
- `/tools`：Agent 工具页
- `/help`：帮助中心页
