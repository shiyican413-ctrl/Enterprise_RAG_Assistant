# 部署与启动说明

## 当前版本

当前已完成第一版 AI 服务 MVP：

- FastAPI 服务
- 文档上传接口
- 本地文本切分
- 本地知识库持久化
- 轻量词频相似度检索
- RAG 问答接口
- SSE 流式接口
- 引用来源返回
- 会话历史保存
- GLM Embedding-3 向量检索
- Next.js 前端控制台
- 文档上传、问答、知识库列表、引用来源展示

当前版本不强依赖外部模型 API Key，方便先验证完整链路。配置 `GLM_API_KEY` 后，文档入库和问答检索会使用 GLM `embedding-3`；未配置时自动降级到本地稀疏检索。后续可以把 `backend/ai_service/services/rag_service.py` 中的模板生成替换为通义千问、Ollama 或 OpenAI Compatible API。

## 安装依赖

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

## 启动服务

```bash
uvicorn backend.ai_service.main:app --reload --host 127.0.0.1 --port 8000
```

推荐在项目根目录创建本地 `.env` 配置 GLM Embedding：

```text
GLM_API_KEY=your-glm-api-key
GLM_EMBEDDING_URL=https://open.bigmodel.cn/api/paas/v4/embeddings
GLM_EMBEDDING_MODEL=embedding-3
GLM_CHAT_URL=https://open.bigmodel.cn/api/paas/v4/chat/completions
GLM_FAST_MODEL=GLM-4V-Flash
GLM_THINKING_MODEL=GLM-4.1V-Thinking-Flash
```

当前使用 PostgreSQL + pgvector，继续加入：

```text
DATABASE_URL=postgresql://postgres:postgres@127.0.0.1:5432/enterprise_rag
```

数据库需要提前创建，并安装 pgvector 扩展。服务启动时会自动创建 `documents`、`document_chunks`、`chat_turns` 三张表。上传入库需要配置 `GLM_API_KEY`，用于生成 pgvector 检索所需的 embedding。

如果知识库是在未配置 `GLM_API_KEY` 时导入的，建议配置 Key 后调用 `/api/knowledge/rebuild` 重新生成向量。

启动后访问：

- 健康检查：`http://127.0.0.1:8000/health`
- Swagger 文档：`http://127.0.0.1:8000/docs`

## 启动前端

```bash
cd frontend/next-web
npm install
npm run dev
```

启动后访问：

- 前端控制台：`http://127.0.0.1:3000`

前端默认请求后端：

```text
http://127.0.0.1:8000
```

如需调整，可以创建 `.env.local`：

```bash
NEXT_PUBLIC_API_BASE_URL=http://127.0.0.1:8000
```

## 导入示例文档

```bash
curl -X POST "http://127.0.0.1:8000/api/documents/upload" ^
  -F "file=@data/sample_policy.txt"
```

## 提问

```bash
curl -X POST "http://127.0.0.1:8000/api/chat/ask" ^
  -H "Content-Type: application/json" ^
  -d "{\"question\":\"报销多久可以打款？\"}"
```

返回结果会包含：

- `answer`：回答正文
- `sources`：引用来源
- `conversation_id`：会话 ID
- `answer_mode`：回答模式，`fast` 为快速模式，`thinking` 为思考模式
- `model`：本次调用的 GLM 模型

## API 列表

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/health` | 健康检查 |
| POST | `/api/documents/upload` | 上传并入库文档 |
| GET | `/api/documents` | 查看知识库文档 |
| DELETE | `/api/documents/{document_id}` | 删除文档及片段 |
| POST | `/api/knowledge/rebuild` | 根据 `data/uploads` 重建知识库 |
| POST | `/api/chat/ask` | 普通问答 |
| POST | `/api/chat/stream` | SSE 流式问答，逐段返回模型生成内容 |
| GET | `/api/chat/conversations/{conversation_id}` | 获取会话历史 |

## 流式问答

前端默认调用 `/api/chat/stream`。后端会使用 GLM Chat Completions 的 `stream=true`，收到模型片段后立即通过 SSE 返回：

```text
answer_delta -> answer_delta -> sources -> done
```

- `answer_delta`：模型实时生成的回答片段
- `sources`：本次回答引用的知识库片段
- `done`：会话 ID、回答模式和模型信息

## 回答模式

| 模式 | 请求值 | 模型 | 说明 |
| --- | --- | --- | --- |
| 快速模式 | `fast` | `GLM-4V-Flash` | 更快返回，适合常规制度查询和简单问答 |
| 思考模式 | `thinking` | `GLM-4.1V-Thinking-Flash` | 更强调综合分析，适合跨片段、多条件问题 |
