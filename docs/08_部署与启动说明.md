# 部署与启动说明

## 当前版本

当前已完成第一版 AI 服务 MVP：

- FastAPI 服务
- 文档上传接口
- 本地文本切分
- 本地知识库持久化
- 轻量词频相似度检索
- RAG 问答接口
- SSE 流式接口
- 引用来源返回
- 会话历史保存
- Next.js 前端控制台
- 文档上传、问答、知识库列表、引用来源展示

第一版不强依赖外部大模型 API Key，方便先验证完整链路。后续可以把 `backend/ai_service/services/rag_service.py` 中的模板生成替换为通义千问、Ollama 或 OpenAI Compatible API。

## 安装依赖

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

## 启动服务

```bash
uvicorn backend.ai_service.main:app --reload --host 127.0.0.1 --port 8000
```

启动后访问：

- 健康检查：`http://127.0.0.1:8000/health`
- Swagger 文档：`http://127.0.0.1:8000/docs`

## 启动前端

```bash
cd frontend/next-web
npm install
npm run dev
```

启动后访问：

- 前端控制台：`http://127.0.0.1:3000`

前端默认请求后端：

```text
http://127.0.0.1:8000
```

如需调整，可以创建 `.env.local`：

```bash
NEXT_PUBLIC_API_BASE_URL=http://127.0.0.1:8000
```

## 导入示例文档

```bash
curl -X POST "http://127.0.0.1:8000/api/documents/upload" ^
  -F "file=@data/sample_policy.txt"
```

## 提问

```bash
curl -X POST "http://127.0.0.1:8000/api/chat/ask" ^
  -H "Content-Type: application/json" ^
  -d "{\"question\":\"报销多久可以打款？\"}"
```

返回结果会包含：

- `answer`：回答正文
- `sources`：引用来源
- `conversation_id`：会话 ID

## API 列表

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/health` | 健康检查 |
| POST | `/api/documents/upload` | 上传并入库文档 |
| GET | `/api/documents` | 查看知识库文档 |
| DELETE | `/api/documents/{document_id}` | 删除文档及片段 |
| POST | `/api/knowledge/rebuild` | 根据 `data/uploads` 重建知识库 |
| POST | `/api/chat/ask` | 普通问答 |
| POST | `/api/chat/stream` | SSE 流式问答 |
| GET | `/api/chat/conversations/{conversation_id}` | 获取会话历史 |
