# 知识库管理页面功能清单

本文档用于规划 `Enterprise RAG Assistant` 的独立知识库管理页面。当前系统已经具备文档上传、入库、列表、删除、重建索引和 RAG 引用返回能力；下一步应把右侧知识库面板升级为独立页面，让知识维护人员可以稳定管理企业资料、检查入库质量、定位检索问题。

## 1. 页面定位

知识库管理页面不是聊天页的附属侧栏，而是企业资料的运维工作台。它的核心目标是回答四个问题：

- 知识库里现在有什么资料？
- 每份资料是否成功解析、切分、向量化并可检索？
- 某份资料命中了哪些问答，质量是否可靠？
- 管理员如何安全地上传、删除、重建和排查资料？

建议路由：

- `/knowledge`：知识库概览与文档列表
- `/knowledge/upload`：文档上传与入库任务
- `/knowledge/documents/[documentId]`：文档详情、片段、引用记录
- `/knowledge/tasks`：入库任务与索引任务记录

第一期可以先做 `/knowledge` 单页，将上传、列表、详情抽屉、索引操作放在一个页面内；等功能稳定后再拆成多路由。

## 2. 用户角色

| 角色 | 主要诉求 | 页面权限 |
| --- | --- | --- |
| 普通员工 | 知道知识库是否有资料，不直接维护 | 只读概览，可查看公开文档摘要 |
| 知识库维护员 | 上传、更新、删除资料，检查入库质量 | 文档上传、删除、重建、查看片段 |
| 管理员 | 管理全局资料、权限、审计和索引策略 | 全量管理、任务日志、权限范围、危险操作 |

当前 MVP 暂未实现登录和 RBAC，可以先在前端保留权限入口和禁用态文案，后续接入 Django 业务后台。

## 3. 页面信息架构

### 3.1 顶部区域

内容：

- 页面标题：知识库
- 副标题：企业资料、索引状态与引用质量管理
- 全局搜索框：按文档名、标签、部门、上传人搜索
- 主要操作按钮：上传文档、刷新、重建索引
- 服务状态：后端在线、向量库在线、Embedding 可用

交互：

- 点击上传文档，打开上传面板或进入 `/knowledge/upload`
- 点击刷新，重新请求文档列表和统计数据
- 点击重建索引，弹出确认框，说明会清空并重新生成当前知识库索引

### 3.2 统计卡片

建议展示 4 到 6 个指标：

- 文档总数
- 片段总数
- 今日新增文档
- 索引成功率
- 最近一次重建时间
- 最近 7 天引用命中次数

第一期可基于现有 `/api/documents` 聚合出文档总数和片段总数；其余指标需要后端扩展任务表、审计表或引用日志。

### 3.3 文档列表

字段建议：

| 字段 | 说明 | 第一期是否需要 |
| --- | --- | --- |
| 文档名称 | 原始文件名 | 是 |
| 类型 | `pdf`、`md`、`txt`、`csv`、`json` | 是 |
| 片段数 | 切分后的 chunk 数量 | 是 |
| 入库状态 | 待处理、处理中、成功、失败 | 建议预留，第一期可默认为成功 |
| 上传时间 | 文档创建时间 | 是 |
| 上传人 | 后续接入用户系统 | 后续 |
| 文件大小 | 原始文件大小 | 建议补充 |
| MD5 | 用于去重和版本判断 | 建议补充 |
| 标签/部门 | 文档分类与权限过滤 | 后续 |
| 操作 | 查看、删除、重新入库、复制 ID | 是 |

列表能力：

- 分页或虚拟滚动
- 按文档名称搜索
- 按文件类型筛选
- 按入库状态筛选
- 按上传时间排序
- 多选批量删除
- 空状态引导上传企业资料

第一期建议优先实现搜索、类型筛选、删除和详情查看，分页可以在文档量增大后补。

### 3.4 上传与入库

上传能力：

- 支持 `.txt`、`.md`、`.csv`、`.json`、`.pdf`
- 支持单文件上传
- 显示文件名、大小、类型校验结果
- 上传后自动完成解析、切分、Embedding 和写库
- 成功后刷新文档列表
- 失败时展示原因，例如格式不支持、解析失败、Embedding 服务不可用

二期增强：

- 批量上传
- 拖拽上传
- 重复文件检测
- 大文件异步任务
- 失败重试
- 上传前填写部门、标签、权限范围、业务分类

### 3.5 文档详情

点击文档行后打开右侧抽屉或详情页。

详情内容：

- 基础信息：文档 ID、名称、类型、上传时间、片段数、MD5、存储路径
- 入库链路：保存原文件、解析文本、切分片段、生成向量、写入向量库
- 文本片段列表：片段序号、内容预览、字符数、Embedding 模型
- 操作记录：上传、删除、重建、失败重试
- 引用记录：哪些问答命中过该文档，命中分数是多少

第一期可先做基础信息和片段预览；引用记录需要新增问答引用日志。

### 3.6 片段管理

片段是 RAG 质量排查的关键视角。

建议能力：

- 查看某文档全部片段
- 搜索片段内容
- 查看片段序号和字符长度
- 查看片段来源元数据
- 复制片段内容
- 后续支持单片段禁用、重新切分、人工修订

第一期后端还没有独立片段查询接口，建议新增：

- `GET /api/documents/{document_id}/chunks`
- `GET /api/chunks/{chunk_id}`

### 3.7 索引操作

操作项：

- 刷新列表：重新获取文档数据
- 删除文档：删除 `documents` 和关联 `document_chunks`
- 重新入库：基于原始文件重新解析并生成向量
- 重建知识库：清空并从 `data/uploads` 全量重建

危险操作要求：

- 删除和重建必须二次确认
- 确认框说明影响范围
- 操作完成后展示结果
- 后续写入审计日志

当前后端已有：

- `DELETE /api/documents/{document_id}`
- `POST /api/knowledge/rebuild`

前端还需要把删除和重建按钮从文档能力中显式暴露出来。

### 3.8 检索测试

知识库管理页建议提供一个轻量“检索测试”面板，不直接生成大模型回答，只检查召回质量。

输入：

- 测试问题
- `top_k`
- 相似度阈值
- 可选文档类型、部门、标签过滤

输出：

- 命中文档
- 命中片段
- 匹配度分数
- 片段内容
- 是否低于阈值

建议新增接口：

- `POST /api/knowledge/search`

这个能力能快速解释“为什么聊天页回答说知识不足”。

### 3.9 空状态与异常状态

空状态：

- 无文档：展示上传入口和支持格式
- 无搜索结果：提示调整关键词或筛选条件
- 无片段：提示文档可能解析失败或尚未入库

异常状态：

- 后端离线：禁用上传和重建，只保留页面骨架
- Embedding 不可用：上传按钮禁用或上传后显示入库失败
- 数据库不可用：提示联系管理员
- 重建中：阻止重复重建，显示任务进度

## 4. 第一期功能边界

建议第一期做成“可用的知识库运维台”，范围不要过大。

必须完成：

- 独立知识库页面 `/knowledge`
- 文档统计：文档数、片段数、状态
- 文档上传
- 文档列表
- 文档删除
- 刷新列表
- 重建索引
- 搜索文档名
- 文件类型筛选
- 基础空状态、加载态、错误态

建议完成：

- 文档详情抽屉
- 入库链路展示
- 文档 MD5、扩展名、原始路径展示
- 删除和重建二次确认

后续再做：

- 批量上传
- 任务队列和进度
- 文档版本管理
- 权限过滤
- 引用日志
- 检索测试
- 片段人工修订
- 混合检索和 Rerank 参数配置

组件职责：

- `rebuild-index-dialog.tsx`：重建确认和结果展示

## 6. API 清单

当前已有接口：

| 方法 | 路径 | 用途 |
| --- | --- | --- |
| GET | `/api/documents` | 获取文档列表 |
| POST | `/api/documents/upload` | 上传并入库文档 |
| DELETE | `/api/documents/{document_id}` | 删除文档及片段 |
| POST | `/api/knowledge/rebuild` | 从上传目录重建知识库 |
| GET | `/health` | 服务健康检查 |

建议新增接口：

| 方法 | 路径 | 用途 |
| --- | --- | --- |
| GET | `/api/knowledge/stats` | 获取文档数、片段数、索引状态等统计 |
| GET | `/api/documents/{document_id}` | 获取文档详情 |
| GET | `/api/documents/{document_id}/chunks` | 获取文档片段 |
| POST | `/api/documents/{document_id}/reingest` | 单文档重新入库 |
| POST | `/api/knowledge/search` | 检索测试，不生成大模型回答 |
| GET | `/api/knowledge/tasks` | 查看入库和重建任务 |

## 7. 验收标准

页面级验收：

- 进入知识库页面后能看到统计、文档列表和上传入口
- 上传支持格式文件后，列表自动出现新文档和片段数
- 上传不支持格式时，页面给出明确错误提示
- 删除文档后，列表和统计同步更新
- 重建索引需要二次确认，完成后显示结果
- 搜索和筛选不会造成页面卡死或布局错乱
- 后端离线时页面能显示服务不可用状态

数据级验收：

- 文档列表数量与数据库 `documents` 表一致
- 片段总数与 `document_chunks` 表一致
- 删除文档会级联删除对应片段
- 重建后文档和片段能被聊天页正常引用

体验级验收：

- 页面重点是资料管理，不混入聊天主流程
- 维护员能在 30 秒内完成上传、检查、删除三个常用动作
- 危险操作都有明确反馈和确认
- 空知识库状态下能自然引导用户上传第一份资料

## 8. 推荐实现顺序

1. 把侧栏里的文档统计、上传和文档列表迁移到独立 `/knowledge` 页面。
2. 补齐前端 `deleteDocument`、`rebuildKnowledge` API 封装。
3. 增加文档列表搜索、类型筛选和操作按钮。
4. 增加删除、重建确认弹窗和成功/失败提示。
5. 增加文档详情抽屉，展示 metadata 和处理链路。
6. 后端补 `GET /api/documents/{document_id}/chunks`，让详情页能看片段。
7. 再扩展任务表、引用日志、权限和检索测试。
