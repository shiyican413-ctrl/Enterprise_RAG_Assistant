# Enterprise RAG Assistant 项目架构与功能方案

## 1. 项目定位

`Enterprise RAG Assistant` 是一个面向企业内部知识管理、客服辅助、制度查询、产品资料问答等场景的知识库智能问答平台。

系统基于企业内部文档构建知识库，支持文档上传、文本切分、向量化检索、RAG 问答、多轮会话、引用溯源和 Agent 工具调用，并预留 Dify 低代码方案和 CrewAI 多智能体扩展方案。

项目目标不是简单 Demo，而是整理成一个具备工程架构、前后端分离、AI 服务解耦、可扩展能力的企业级 AI 应用项目。

## 2. 适用业务场景

- 企业制度问答：报销制度、请假制度、入职流程、IT 支持流程。
- 产品资料问答：产品参数、使用说明、适用场景、售前 FAQ。
- 售后客服辅助：故障排查、退换货政策、操作手册查询。
- 内部培训助手：员工基于培训资料进行问答学习。
- 电商导购助手：尺码推荐、颜色搭配、洗涤养护、搭配建议。
- 业务流程辅助：结合 Agent 工具调用查询订单、库存、工单、员工信息等。

## 3. 技术架构

```text
Next.js + TypeScript
企业前端 / 聊天页面 / 知识库管理 / 管理后台
        |
      Django

        |
      FastAPI
```

## 4. 技术选型说明

### 4.1 前端

- `Next.js`：用于构建正式企业 Web 应用，支持路由、服务端渲染、组件化页面。
- `TypeScript`：提升前端代码类型安全和可维护性。
- `React`：构建聊天 UI、知识库管理、上传页面、配置页面。

前端主要负责：

- 登录页面
- 智能问答页面
- 知识库管理页面
- 文档上传页面
- 引用来源展示
- 历史会话记录
- 模型配置页面
- 管理员后台页面

### 4.2 业务后端

- `Django`：负责传统企业后台能力。
- `Django ORM`：管理用户、角色、文档元数据、操作日志等结构化数据。
- `RBAC`：实现管理员、普通用户、知识库维护人员等角色权限。

Django 主要负责：

- 用户登录与认证
- 角色权限管理
- 文档元数据管理
- 操作日志记录
- 后台管理页面
- 与 FastAPI AI 服务通信

### 4.3 AI 服务层

- `FastAPI`：负责 AI 能力接口，适合构建高性能 API 和流式响应接口。
- `LangChain`：负责 RAG 链路、Prompt、Retriever、Memory、Chain。
- `LangGraph`：用于后续更复杂的 Agent 状态流转和任务编排。

FastAPI 主要负责：

- 文档向量化接口
- 知识库检索接口
- RAG 问答接口
- SSE/WebSocket 流式输出
- Agent 工具调用接口
- 模型服务适配

### 4.4 向量数据库

第一版使用：

- `Chroma`：适合本地开发和快速验证。

后续可扩展：

- `pgvector`：适合接入企业已有 PostgreSQL 系统。
- `Qdrant`：适合更专业的向量检索、过滤和混合检索场景。

### 4.5 模型接入

- `DashScope / 通义千问`：云端大模型和 Embedding 模型。
- `Ollama`：本地模型部署和离线验证。
- `OpenAI Compatible API`：统一适配不同模型服务。

## 5. 核心功能设计

### 5.1 用户与权限管理

- 用户注册与登录
- 管理员和普通用户角色区分
- 知识库访问权限控制
- 用户操作日志记录

### 5.2 知识库管理

- 上传企业文档
- 查看文档列表
- 查看文档处理状态
- 删除文档及对应向量索引
- 重建知识库索引
- MD5 文件去重
- 文档来源、上传人、上传时间等元数据管理

### 5.3 文档处理

第一版支持：

- `.txt`
- `.pdf`
- `.csv`
- `.json`

后续扩展：

- `.docx`
- `.xlsx`
- Markdown
- 网页 FAQ
- OCR 图片文字识别

处理流程：

```text
上传文档
  -> 解析文本
  -> 清洗内容
  -> 文本切分
  -> 生成 Embedding
  -> 写入向量库
  -> 保存文档元数据
```

### 5.4 RAG 智能问答

问答流程：

```text
用户提问
  -> 查询历史会话
  -> 向量检索相关文档片段
  -> 构造 Prompt
  -> 调用大模型
  -> 生成回答
  -> 返回引用来源
  -> 保存会话记录
```

第一版能力：

- 基于向量相似度检索
- 支持多轮会话
- 支持上下文增强生成
- 支持回答引用来源

后续优化：

- 混合检索：向量检索 + 关键词检索
- Rerank 重排序
- 相似度阈值过滤
- 按文档类型、部门、权限做元数据过滤

### 5.5 引用来源展示

回答结果应包含：

- 最终回答内容
- 引用文档名称
- 引用片段
- 文档元数据
- 相似度分数

企业场景中，引用来源非常重要，可以降低模型幻觉风险，并方便用户追溯答案依据。

### 5.6 多轮会话

- 保存用户历史提问和模型回答
- 支持基于上下文继续追问
- 支持按用户、会话 ID 查询历史记录
- 后续可扩展会话标题、收藏、导出等功能

### 5.7 流式输出

FastAPI 提供 SSE 或 WebSocket 接口，Next.js 前端实现类似 ChatGPT 的打字机式输出。

用途：

- 提升用户体验
- 适合长回答生成
- 方便观察模型响应过程

### 5.8 Agent 工具调用

将企业业务接口封装为工具，交给 Agent 调用。

示例工具：

- 查询订单状态
- 查询库存
- 查询员工假期余额
- 查询报销进度
- 创建客服工单
- 查询产品价格

Agent 流程示例：

```text
用户提问
  -> 判断是否需要调用工具
  -> 调用业务 API
  -> 结合知识库检索结果
  -> 生成最终回答
```

## 6. Dify 对照方案

Dify 不作为主线工程实现，而是作为低代码 AI 应用方案进行对照。

规划内容：

- 使用 Dify 创建知识库
- 上传同一批企业文档
- 配置知识库问答应用
- 配置工作流和 Agent
- 发布聊天应用
- 对比 Dify 方案和自研 LangChain 方案

对比维度：

- 开发效率
- 灵活性
- 可控性
- 私有化部署
- 业务系统集成能力
- 后续工程扩展能力

简历表达建议：

> 调研 Dify 知识库、工作流和 Agent 能力，搭建低代码版本原型，对比自研 LangChain 方案在灵活性、可控性和工程化方面的差异。

## 7. CrewAI 多智能体扩展方案

CrewAI 作为二期扩展，不建议第一版重度实现。

可设计如下多 Agent 流程：

```text
分类 Agent
  -> 判断问题属于 HR、财务、产品、IT 或售后

检索 Agent
  -> 从对应知识库中检索相关资料

回答 Agent
  -> 根据检索结果生成正式回答

审核 Agent
  -> 检查回答是否基于知识库，是否存在无依据内容

工单 Agent
  -> 当资料不足时，引导转人工或创建工单
```

适合企业复杂问题处理场景，也适合作为项目高级亮点。

## 9. 实现优先级

### 第一阶段：RAG 核心链路

- 整理现有 P4 RAG 项目代码
- 修复完整 RAG 链路
- 实现文档上传、切分、向量化、检索、问答
- 使用 Chroma 作为本地向量库
- 接入通义千问或 Ollama

### 第二阶段：FastAPI AI 服务

- 封装文档上传接口
- 封装知识库入库接口
- 封装问答接口
- 支持流式输出
- 返回引用来源

### 第三阶段：Next.js 前端

- 实现登录页
- 实现知识库管理页
- 实现文档上传页
- 实现智能问答页
- 实现引用来源展示
- 实现历史会话列表

### 第四阶段：Django 业务后台

- 实现用户认证
- 实现角色权限
- 实现文档元数据管理
- 实现操作日志
- 对接 FastAPI AI 服务

### 第五阶段：企业级增强

- 混合检索
- Rerank 重排序
- pgvector 或 Qdrant 替换方案
- LangGraph Agent 工作流
- Dify 对照文档
- CrewAI 扩展文档

## 10. 项目亮点

- 采用 Next.js + TypeScript 构建正式企业前端，不停留在 Streamlit Demo。
- 采用 Django + FastAPI 分层架构，传统业务系统和 AI 服务解耦。
- 基于 LangChain 实现完整 RAG 流程，覆盖文档切分、向量化、检索、生成。
- 支持引用溯源，提升企业知识库问答可信度。
- 支持多轮会话和流式输出，提升实际交互体验。
- 支持 Agent 工具调用，为企业业务系统集成预留空间。
- 预留 Dify 低代码对照方案，体现对企业 AI 平台落地路径的理解。
- 预留 CrewAI 多智能体扩展方案，体现复杂任务编排能力。

## 11. 简历项目描述

> 基于 Next.js + TypeScript、Django、FastAPI 和 LangChain 构建企业级知识库智能问答平台，支持文档上传、文本切分、向量化检索、RAG 问答、多轮会话、引用溯源和 Agent 工具调用。系统采用 Django 负责用户权限与业务管理，FastAPI 负责 AI 服务和流式问答，前端使用 Next.js 实现知识库管理和智能问答页面，并调研 Dify 低代码方案与 CrewAI 多智能体扩展，提升企业 AI 应用的可落地性与扩展性。

## 12. 面试讲解思路

可以按下面顺序讲：

1. 为什么做这个项目：企业有大量文档，但查找效率低，普通大模型又无法保证基于企业资料回答。
2. 系统怎么解决：通过 RAG 把企业文档向量化入库，用户提问时先检索相关资料，再让模型基于资料回答。
3. 为什么分层：Django 负责用户、权限、文档、日志；FastAPI 负责 AI 服务；Next.js 负责正式前端页面。
4. 项目难点：文档切分、检索准确率、引用来源、多轮会话、流式输出、Agent 工具调用。
5. 后续优化：混合检索、Rerank、LangGraph、pgvector/Qdrant、Dify 对照、CrewAI 多智能体。
